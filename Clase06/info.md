UML significa Unified Modeling Language (Lenguaje Unificado de Modelado). Es un lenguaje visual que se usa para diseñar, planificar y documentar sistemas de software antes de programarlos.

En lugar de escribir código directamente, UML te permite dibujar diagramas que muestran cómo funciona un sistema.

🧩 ¿Para qué sirve?

Se utiliza para:

Entender sistemas complejos
Comunicar ideas entre desarrolladores y equipos
Diseñar la arquitectura de software
Documentar proyectos
📊 Tipos de diagramas más comunes
1. Diagrama de clases

Muestra las clases, sus atributos y relaciones (muy usado en programación orientada a objetos).

2. Diagrama de casos de uso

Describe qué puede hacer el sistema desde la perspectiva del usuario.

3. Diagrama de secuencia

Explica cómo interactúan los objetos en el tiempo (paso a paso).

4. Diagrama de actividades

Representa flujos de trabajo o procesos (similar a un diagrama de flujo).

🧠 Ejemplo simple

Imagina una app de compras:

Usuario → inicia sesión
Usuario → compra producto
Sistema → procesa pago

Eso se puede representar visualmente con UML para que todos entiendan cómo funciona.

🚀 En resumen

UML es como un “plano” del software, igual que los planos en arquitectura, pero para sistemas informáticos.
